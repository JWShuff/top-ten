The a element

So, the a element serves to format links and act as a clickable element on a page. It can point to links, emails, even locations in this document!

It also has all sorts of CSS styling options available that alter behavior from hovered, to clicked, and so on.

 
        < a > is the opening tag and < / a > is the closing tag. < a href= >normally begins the link, and you can use mailto:, #(insertIDInPageHere), or a web link.

The pre marker

I had to look up what the < pre > does in order to write things properly. So now you get to learn too!

It also has all sorts of CSS styling options available that alter behavior from hovered, to clicked, and so on. 

 
        < pre > opens and < /pre > closes the block. No text is formatted in the block...but pre {} in your style sheet allows you to define a set styling for text inside of pre blocks.

Input

So input is almost definitely going to be a thing we use a ton. It's got some predefined "types" where it will do some data validation on your behalf and/or instruct mobile browsers what sorts of inputs to use.

input can also handle numbers, or email addresses. But you shouldn't count on the outputs of the element to be fully validated before use in your programs.
